# Nodes Canvas Agent Context (AGENT.md)

이 문서는 마인드 맵 노드 캔버스 영역 (`/src/app/nodes/`)에 투입되는 다른 AI 에이전트들에게 공유 및 상속될 중요 비즈니스 및 엔지니어링 컨텍스트를 서술합니다.

---

## 1. 캔버스 상태 관리 및 React Flow 바인딩

노드 캔버스는 React Flow 라이브러리를 이용하여 시각 흐름을 조작하며, 모든 노드 데이터와 연결선 정보는 훅과 컴파일러 상에서 아래 명세를 철저히 따릅니다.

- **노드 상태 (`nodes`)**: `useNodesState`를 사용하여 상태 관리합니다. 각 노드는 고유 ID(`id`)를 보유하며, 커스텀 디자인을 적용하기 위해 반드시 `type: 'neo'`로 설정되어야 합니다.
- **에지 상태 (`edges`)**: `useEdgesState`를 사용하여 연결선 정보를 관리합니다.
- **커넥션 룰 (`onConnect`)**:
  - 두 노드 간의 조인트 드래그 오버 시 트리거되는 `onConnect` 콜백 함수 내에서 생성되는 새로운 에지 또한 글래스모피즘 시그니처 톤을 잃지 않도록 테마 스타일링이 강제 세팅되어 있습니다.
  - 생성 엣지 속성: `style: { strokeWidth: 3, stroke: 'rgba(14, 165, 233, 0.4)' }, type: 'smoothstep'`

---

## 2. 다른 에이전트 주의 및 금지 사항

### 1) 숨김 컴포넌트 (`_components/`) 확장 의무
- **규격**: Nodes 페이지 하위에서만 독립 활용되는 컴포넌트는 전역 `src/components/ui` 에 절대 난입시켜서는 안 되며, 반드시 이 로컬 디렉토리의 프라이빗 숨김 폴더인 `_components/` 하위에 격리하여 생성해야 합니다.
- **예시**: 커스텀 노드 `NeoNode`는 `src/app/nodes/_components/NeoNode/` 하위에 배치되어 관리됩니다.

### 2) React Flow 버전 및 모듈 임포트 안정성
- `ReactFlow` 컴포넌트 하위에 배경 점 패턴 렌더러를 얹을 때, 컴파일러 버전에 따라 `BackgroundVariant`가 제대로 바인딩되지 않는 현상이 발생할 수 있습니다. 
- 이를 예방하기 위해 페이지 하단에 `BackgroundVariant` 열거형 상수를 플레이스홀더 성격으로 수동 매핑해 두었으니 임의로 교체하거나 무단 제거하여 컴파일을 깨뜨리지 마십시오.
