# Funny Todo Agent Guidebook (AGENT.md)

이 문서는 Funny Todo 프로젝트에 투입되는 모든 AI 에이전트가 완벽히 숙지하고 따라야 하는 중요 비즈니스 컨텍스트, 시스템 아키텍처 규칙, 그리고 개발 제약 사항을 담은 마스터 가이드북입니다. 본 가이드북을 거치지 않고 소스 코드를 임의 변경하는 행위는 엄격히 금지됩니다.

---

## 1. 프로젝트 개요 및 핵심 비즈니스 도메인

Funny Todo는 단순한 할 일 관리 애플리케이션을 넘어, 창의적인 영감과 계획들을 상호 연관된 흐름망으로 시각화하고 실행하는 **아이디어 체인 빌더(Idea Chain Builder)** 서비스입니다.

### 핵심 비즈니스 로직: 아이디어 선행조건 잠금 해제 (Prerequisite Cascade Unlock)
1. **잠금 상태 (`locked`)**: 특정 아이디어는 하나 이상의 선행 아이디어(Prerequisites)를 가집니다. 선행 조건이 충족되지 않은 아이디어는 사용자가 임의로 완료하거나 편집할 수 없도록 강한 잠금 상태로 렌더링되어야 합니다.
2. **실행 가능 상태 (`ready`)**: 연결된 모든 선행 아이디어가 '완료(smashed)' 상태가 되는 순간, 시스템은 즉시 캐스케이딩(Cascading) 규칙에 따라 해당 아이디어를 실행 가능한 '준비(ready)' 상태로 자동 전환해야 합니다.
3. **완료 상태 (`smashed`)**: 사용자가 실행을 완료하여 준비된 아이디어를 타격(smash)하여 달성한 최종 상태입니다.

### 마인드 그래프 캔버스 (Nodes & Edges Canvas)
- 사용자의 비주얼 맵 구성을 위해 React Flow 모듈을 탑재하고 있습니다.
- 개별 아이디어들은 **노드(Node)**로 나타나며, 아이디어 사이의 인과 관계나 선행 및 후속 연결 고리들은 **엣지(Edge)**로 연결되어 유기적으로 연동되어야 합니다.

## 2. 에이전트 필수 준수 규칙 (Global Rules)

### 1. 언어 및 대화 원칙
- **한국어 기본 사용**: 모든 문서와 대화, 안내는 정중하고 자연스러운 한국어를 채택합니다. 번역투의 딱딱한 문체는 지양합니다.
- **이모티콘 사용 금지**: 정보의 격조와 일관성을 보존하기 위해 보고서, 로그, 대화 상에서 이모티콘 기호(예: 😊, 💡, 🕸️ 등)의 노출을 일절 배제합니다.
- **간결성과 시각적 계층화**: 중복 서술을 전격 배제하고, 마크다운 표와 체크리스트를 적극적으로 활용하여 한눈에 들어오는 계층 구조를 만듭니다.

### 2. WSL 내부 우회 실행 수칙 (WSL-Safe Command Execution)
- **파일 시스템 락 대응**: Windows 호스트 환경에서 프로젝트 마운트 영역(`z:\home\minhulee\Projects\funny-todo`)을 직접 다룰 경우, 이종 운영체제 간 파일 충돌로 인한 권한 및 입출력 오류(`EPERM`, `EISDIR`)가 주기적으로 발생합니다.
- **안전 실행 프로토콜**: 모든 패키지 설치(`npm install`), 정적 빌드(`npm run build`), 개발 서버 실행(`npm run dev`) 등 터미널 명령을 시도할 때는 반드시 **WSL 내 가상 Ubuntu 경로**로 디렉토리를 전환하여 우회 실행해야 합니다.
  - *올바른 명령어 템플릿*: `wsl --cd /home/minhulee/Projects/funny-todo [원하는 명령어]`

### 3. 타입 세이프티 (TypeScript Type Assertions)
- 인라인 스타일 객체를 HTML 태그 및 모션 요소에 지정할 때 컴파일러의 와일드카드 타입 추론을 교정하기 위해, 바인딩 시 반드시 `as React.CSSProperties` 타입 단언을 선언적으로 표기합니다.
- `useIdeas` 훅은 컴포넌트 렌더링에 안정적인 형식을 제공하기 위해 `RelationType` 타입을 상시 명시하고 내보냅니다.

## 3. 아키텍처 및 디자인 연동 구조

프로젝트의 디자인 고도화는 다음 두 파일의 명세를 토대로 일관성 있게 추진해야 합니다.

1. **전역 디자인 규격**: [.agents/DESIGN.md](file:///.agents/DESIGN.md)
   - 전반적인 글래스모피즘 테마 설정과 섀도우, 연한 블루 디자인 토큰이 수록되어 있습니다.
2. **아토믹 파일 규격**: `/src/ui/` 하위 아토믹 폴더
   - `/index.tsx` (컴포넌트)
   - `/style.js` (스타일링 객체)
   - 컴포넌트의 디자인 사양은 개별 `.pen` 파일 대신 **[funny-todo.pen](file:///z:/home/minhulee/Projects/funny-todo/.agents/funny-todo.pen)** 단일 디자인 프로젝트 파일에 병합하여 통합 관리 및 동기화합니다.
